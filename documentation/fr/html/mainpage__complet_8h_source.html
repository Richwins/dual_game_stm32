2 * @mainpage Projet SYS3046 – Jeu de Duel STM32

4 * @section intro_sec Présentation générale
5 *
6 * Ce projet regroupe deux sous-systèmes complémentaires qui forment ensemble
7 * un **jeu de duel multijoueur** piloté par des cartes STM32 :
8 *
9 * | Sous-système | Technologie | Rôle |
10 * |---|---|---|
11 * | @ref grp_micro "Firmware STM32" | C / STM32L053 | Lecture joystick, LEDs de vie, communication UART |
12 * | @ref grp_jeu "Interface Web" | Vue.js / Node.js | Logique de jeu, rendu, pont WebSocket ↔ série |
13 *
14 * @section ui_sec Aperçu de l'interface
15 *
16 * @image html ui.png "Interface du jeu de duel – Vue en cours de partie"
17 * @image latex ui.png "Interface du jeu de duel" width=8cm
18 *
19 * L'interface affiche :
20 * - **En haut** : HUD des deux joueurs (nom, barre de vie, LEDs, score) + chronomètre central
21 * - **Au centre** : Arène de jeu avec les deux avions et la zone neutre
22 * - **En bas** : Contrôles clavier, boutons de gestion et journal des événements

75 * @section rules_sec Règles du jeu
76 *
77 * - Chaque joueur commence avec **30 points** (= 3 LEDs allumées).
78 * - Chaque touche adverse retire **1 point**.
79 * - Quand le score passe sous un seuil de 10 : **une LED s'éteint**.
80 * - Score = 0 → joueur éliminé → fin de manche.
81 * - Appuyer sur le **bouton de tir (1)** après la victoire relance une manche.
82 *

85 * @section hw_sec Matériel et protocole série
86 *
87 * @subsection hw_joystick Joystick analogique
88 *
89 * Le joystick est connecté sur les entrées ADC :
90 * - **PA0 (ADC_IN0)** : axe X
91 * - **PA1 (ADC_IN1)** : axe Y
92 *
93 * Le firmware lit les valeurs ADC (0–4095) et envoie une direction :
94 * | Valeur ADC | Direction envoyée |
95 * |------------|-------------------|
96 * | vx < 1900 | `DOWN\n` |
97 * | vx > 2200 | `UP\n` |
98 * | vy < 1900 | `RIGHT\n` |
99 * | vy > 2200 | `LEFT\n` |
100 *
101 * @subsection hw_button Bouton de tir
102 *
103 * - **PA4** : entrée digitale, actif haut
104 * - Envoie `1\n` quand pressé
105 * - Utilisé pour tirer pendant le jeu ou relancer après une victoire
106 *
107 * @subsection hw_leds LEDs de vie
108 *
109 * Trois LEDs représentent les vies du joueur :
110 * | LED | Broche | Seuil de points |
111 * |-------|--------|-----------------|
112 * | LED 1 | PB6 | 21–30 pts |
113 * | LED 2 | PC7 | 11–20 pts |
114 * | LED 3 | PA9 | 1–10 pts |
115 *
116 * Commande reçue : **'L'** → éteint la prochaine LED (avec clignotement)
117 *
118 * @subsection hw_buzzer Buzzer
119 *
120 * - **PC0** : sortie digitale (buzzer ou LED de signalisation)
121 * - Commande reçue : **'B'** → clignotement 3× puis reset des LEDs
122 * - Utilisé en fin de manche pour signaler la victoire
123 *
124 * @subsection hw_timer Chronomètre (Joueur 2 uniquement)
125 *
126 * - Le STM32 du joueur 2 envoie automatiquement **'T\n'** toutes les secondes
127 * - Utilise la fonction `timeur()` pour mesurer le temps écoulé
128 * - Le temps total s'affiche en haut de l'écran au format `HH:MM:SS`
129 *
130 * @subsection hw_proto Récapitulatif du protocole
131 *
132 * **STM32 → PC (messages envoyés via UART) :**
133 * | Message | Signification |
134 * |-----------|-----------------------------------|
135 * | `UP\n` | Joystick vers le haut |
136 * | `DOWN\n` | Joystick vers le bas |
137 * | `LEFT\n` | Joystick vers la gauche |
138 * | `RIGHT\n` | Joystick vers la droite |
139 * | `1\n` | Bouton de tir pressé |
140 * | `T\n` | Tick chronomètre automatique (joueur 2, chaque seconde) |
141 *
142 * **PC → STM32 (commandes reçues via UART) :**
143 * | Commande | Effet |
144 * |----------|--------------------------------------------|
145 * | `L` | Éteint la prochaine LED (avec clignotement)|
146 * | `B` | Buzzer fin de manche + reset LEDs |

150 * @section install_sec Installation et lancement
151 *
152 * @subsection install_prereq Prérequis
153 *
154 * - **Node.js** v18+ installé ([nodejs.org](https://nodejs.org))
155 * - **npm** (inclus avec Node.js)
156 * - Deux cartes **STM32L053** flashées avec le firmware `Microcontroleurs`
157 * - Câbles USB connectés aux ports série (ex: COM6, COM7)
158 *
159 * @subsection install_server 1. Serveur WebSocket (pont série)
160 *
161 * @code{.sh}
162 * cd jeu/server
163 * npm install # installe serialport, ws
164 * @endcode
165 *
166 * Modifier `server.js` si vos ports COM sont différents :
167 * @code{.js}
168 * const PLAYERS = [
169 * { id: 1, path: "COM6", baudRate: 115200 },
170 * { id: 2, path: "COM7", baudRate: 115200 }
171 * ];
172 * @endcode
173 *
174 * Lancer le serveur :
175 * @code{.sh}
176 * node server.js # affiche "Serveur WebSocket lancé : ws://localhost:3000"
177 * @endcode
178 *
179 * @subsection install_web 2. Interface Web (Vue.js)
180 *
181 * @code{.sh}
182 * cd jeu
183 * npm install # installe Vue, Vite et dépendances
184 * npm run dev # démarre le serveur de développement (http://localhost:5173)
185 * @endcode
186 *
187 * @subsection install_run 3. Jouer
188 *
189 * 1. Brancher les deux cartes STM32 en USB
190 * 2. Lancer `node server.js` dans un terminal
191 * 3. Lancer `npm run dev` dans un autre terminal
192 * 4. Ouvrir **http://localhost:5173** dans le navigateur
193 * 5. Le badge passe à "🟢 STM32 connecté" → prêt à jouer !
