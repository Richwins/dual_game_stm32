SYS3046 Project – STM32 Duel Game

Overview

This project consists of two complementary subsystems that together form a multiplayer duel game controlled by STM32 boards:

Subsystem Technology Role
STM32 Firmware C / STM32L053 Joystick reading, life LEDs, UART communication
Web Interface Vue.js / Node.js Game logic, rendering, WebSocket ↔ serial bridge

Interface Preview

Duel game interface – In-game view

The interface displays:

  • Top: HUD for both players (name, health bar, LEDs, score) + central timer
  • Center: Game arena with both planes and neutral zone
  • Bottom: Keyboard controls, management buttons, and event log

Game Rules

  • Each player starts with 30 points (= 3 LEDs lit).
  • Each hit removes 1 point.
  • When score drops below a threshold of 10: one LED turns off.
  • Score = 0 → player eliminated → end of round.
  • Press the fire button (1) after victory to start a new round.

Hardware and Serial Protocol

Analog Joystick

The joystick is connected to ADC inputs:

  • PA0 (ADC_IN0): X axis
  • PA1 (ADC_IN1): Y axis

The firmware reads ADC values (0–4095) and sends a direction:

ADC Value Direction sent
vx < 1900 DOWN\n
vx > 2200 UP\n
vy < 1900 RIGHT\n
vy > 2200 LEFT\n

Fire Button

  • PA4: digital input, active high
  • Sends 1\n when pressed
  • Used to fire during game or restart after victory

Life LEDs

Three LEDs represent player lives:

LED Pin Point threshold
LED 1 PB6 21–30 pts
LED 2 PC7 11–20 pts
LED 3 PA9 1–10 pts

Command received: 'L' → turns off next LED (with blinking)

Buzzer

  • PC0: digital output (buzzer or indicator LED)
  • Command received: 'B' → blinks 3× then resets LEDs
  • Used at end of round to signal victory

Timer (Player 2 only)

  • Player 2's STM32 automatically sends **'T
    '** every second
  • Uses timeur() function to measure elapsed time
  • Total time displayed at top of screen in HH:MM:SS format

Protocol Summary

STM32 → PC (messages sent via UART):

Message Meaning
UP\n Joystick up
DOWN\n Joystick down
LEFT\n Joystick left
RIGHT\n Joystick right
1\n Fire button pressed
T\n Automatic timer tick (player 2, every second)

PC → STM32 (commands received via UART):

Command Effect
L Turn off next LED (with blinking)
B End of round buzzer + reset LEDs

Installation and Launch

Prerequisites

  • Node.js v18+ installed (nodejs.org)
  • npm (included with Node.js)
  • Two STM32L053 boards flashed with Microcontroleurs firmware
  • USB cables connected to serial ports (e.g., COM6, COM7)

1. WebSocket Server (serial bridge)

cd jeu/server
npm install # installs serialport, ws

Edit server.js if your COM ports are different:

const PLAYERS = [
{ id: 1, path: "COM6", baudRate: 115200 },
{ id: 2, path: "COM7", baudRate: 115200 }
];

Start the server:

node server.js # displays "WebSocket server started: ws://localhost:3000"

2. Web Interface (Vue.js)

cd jeu
npm install # installs Vue, Vite and dependencies
npm run dev # starts development server (http://localhost:5173)

3. Play

  1. Connect both STM32 boards via USB
  2. Run node server.js in a terminal
  3. Run npm run dev in another terminal
  4. Open http://localhost:5173 in browser
  5. Badge shows "🟢 STM32 connected" → ready to play!
